# inputcodex 仓库筹备计划

## 目标

在不导入任何应用源码、不预选最终架构的前提下，建立 `inputcodex` 的本地与 GitHub 公开仓库基线，为后续双仓审计和架构讨论提供稳定入口。

## 已批准范围

- 新项目名称为 `inputcodex`。
- 创建 GitHub 公开仓库。
- 采用与两份参考项目一致的 GNU AGPLv3。
- 建立 `README.md`、`AGENTS.md`、`build.md`、`err.md` 和项目计划文档。
- 创建筹备 Issue。

## 明确排除

- 不克隆、复制或迁移参考仓库源码。
- 不删除或改写参考仓库中的广告实现。
- 不决定桌面框架、前端框架、状态管理或存储方案。
- 不实现 UI、功能或性能优化。

## 验收标准

- `nonononull/inputcodex` 为 GitHub 公开仓库。
- 本地 `main` 跟踪 `origin/main`。
- 项目根目录具备构建与排错文档。
- GitHub Issue `#1` 记录后续讨论边界。
- `git diff --check` 通过，仓库状态可解释。

## 下一轮输入

- 两份参考仓库的源码与提交历史审计。
- 用户期望保留、删除和重做的功能清单。
- 卡顿与功能加载异常的复现路径。
- 目标操作系统、分发方式和兼容范围。
